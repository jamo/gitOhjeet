Materiaalin siirto gittiin ja git komennot

Ihan aluksi kannattaa muutama asetus laittaa kohdalleen gitin asetuksista. Näin commiteissa näkyvät tiedot ovat järkeviä.

git config --global user.name "Nimesi"

git config --global user.email foo.bar@osoite.fi

 

git status

näyttää tulosteen, joka kuvaa git reposi tilaa.

$ git status
# On branch master
# Changes to be committed:
# (use "git reset HEAD <file>..." to unstage)
#
# new file: README
#
# Changed but not updated:
# (use "git add <file>..." to update what will be committed)
#
# modified: Main.java
#

Tämä tulostus kertoo, että Main.javaa on muokattu ja ja että README tiedosto on uusi.

git add -A

Tämä lisää kaikki nykyisessä kansiossasi ja sen alikansioissa olevat muutokset ja velä lisäämättömät tiedostot gitin seurantaan.

Lisätäksesi tietyt tiedostot git:in hallittavaksi komenna git add <file> <file2>.

git commit -m "viesti"

Seuraavaksi muutokset tulee committaa. git commit -m "viesti, mitä on tehty"

git push

Nyt muutokset ovat paikallisesti gitissä, mutta vielä kukaan muu palvelimelta tai githubissa ei näe muutoksiasi. Ne täytyy vielä pushata 'pilveen'. Se tapahtuu komennolla git push Kuitenkin ensimmäisellä kerralla joudut komentamaan git push origin master (mikäli lisäsit remote peron git add remote nimi url komennolla korvaa origin nimellä.)

Nyt kaikki muutoksesi on 'pilvessä'.

git pull

Mikäli työskentelet useammalta koneelta samaan repoon tai joku muu on tehnyt muutoksia repon joudut lataamaan muutokset ennen kuin voi pushata mitään. Tämä tapahtuu komennolla git pull.

git

Komentona git tulostaa selostuksen yleisimmin tarvittavista komennoista. ja git help komentonäyttää komennon käyttöohjeen.
