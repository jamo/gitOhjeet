Git ohjeita:

Tämä on lyhyt ohjeistus gitin käytön aloittamiseen ja sisältää perusohjeet gitin käyttöön.

 

Miksi git?

Git on kokemuksieni mukan monipuolisempi ja lopulta helppoikäyttöisempi kuin svn (Subersion). Ensimmäiset 3h saatat kuitenkin olla asiasta eri mieltä, mutta pitempään molempia käyttäneenä suosittelen vain Git:in käyttöä.

GITin käyttöönotto

Gitin käyttöön on käytännössa kaksi tapaa. Joko itse luotu git repositorio tai Github.com.

 

1. Github.com


Github.com tarjoaa erinomaisen käyttöliittymän git:ille ja githubissa tiedostojen selailu on erittäin helppoa. Lisäksi github tarjoaa helpon tavan jakaa repository vain tietyille henkilöille tai koko maailmalle.

2. Git paikallisesti

Voit käyttää git:iä joko paikallisesti yhdelle kansiolla siältöineen, tai käyttää git:iä github tyyppisesti luomalla git repositorion palvelimelle jonne pääsetesimerkiksi ssh-yhteyden avulla. Tällaisessa tapauksessa ona vaikeampaa antaa kaverillesi oikeus katsoa projektejasi.

TKTL laitoksella opiskelevia, jotka eivät halua githubia käytää suosittelen luomaan repositorion users.cs.helsinki.fi -palvelimelle. Siellä kotihakemistosi paikka on aina vakio /home/<käyttäjätunnus>/ kun taas muualla se seilailee levypalvelimella ja polku kotihakemistoosi voi muuttua.
