Päivittäiset git komennot

 

Kun repo on saatu tehtyä ja kloonattua on toiminta helpompaa.

git pull

Aina kun aloitat työskentelyn

 

git add -A

ja

git commit -m "viesti"

Kun olet tehnyt muutoksia ja haluat committaa ne

 

git push

Kun haluat saada työsi päivittymään 'pilveen'

 

Muita käytännöllisiä toimintoja

 

git log --graph --oneline --all

Näyttää kuvan siitä, miten commitit on tehty, ja jos jos ja kun on ollut useampia samanaikaisia versioita eripaikoissa väet miten ne ovat yhdistyneet. Tässä näkyvät myös commit viestit.

 

git tag -a v1.2 9fasb52

Tageilla voi helposti merkata tietyn tilanteen, jolloin sen version palauttaminen on helpompaa, ja voit githubista ladata kyseisen taigin omaavat tiedot zip tiedostona.
Komennon rakenne on seuraava git tag -a <versio> <pala (alku) git commitista syntyneestä checksum:ista.>

 

git push --tags

Tagit pitää kuitenkin erikseen push:ata.
