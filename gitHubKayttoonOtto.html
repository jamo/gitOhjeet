Rekisteröityminen Githubiin

Rekisteröidy githubiin osoitteessa https://github.com/ ja valitse "Get free account" Jos olet opiskelija on suositeltavaa rekisteröityä käyttämällä oppilaitoksen sähköpostiosoitetta, jollon voit helpommin halea ilmaista micro plan:ia jolloin saat viisi ilmaista yksityis repoa.

Jos olet opiskelija käy vielä osoitteessa https://github.com/edu ja kerro heille olevasi opiskelija ja miksi tarvitsisit githubista maksullisen version ilmaiseksi. Hyviksi syiksi voi mainita vaikka tarpeen ohjelmointiprojektien keskitetyyn hallintaan ja koulun (ohjelmoini)ryhmätöiden ryhmätyöskentlyyn. Muista, perustelut pitää tietysti tehdä englanniksi.

Repositorion luominen Githubiin

Valitse new repository, täytä tarvittavat tiedot ja vaitse create repository.

Jotta voit tuottaa materiaalia repositorioosi, tulee sinun lisätä ssh-avain githubiin. Voit luoda uuden ssh avaimen helposti komenolla:
ssh-keyden -t rsa -C "Sinun.sähköpostiosoite@jotain.com" ;
Kun tallennettavan kohteen nimeä kysytään, voit valita siihen minkä haluat tai käyttää oletuksena ehdoitettua. Ubuntussa kuitenkin on suositeltavaa käyttää polkuna ~/.ssh Näin git löytää todennäköisemmin avaimesi.

Komento luo kaksi tiedostoa; oletuksena nimiltään id_rsa ja id_rsa.pub. Haluat avata id_rsa.pub tiedoston tekstieditorissa ja kopioida tiedoston sisällön githubiin.

Githubissa mene Asetukset -> SSH Public Keys https://github.com/account/ssh ja valitse Add another public key Syötä sille otsikko ja kopioi avain Key kenttään.

Nyt olet valmis kloonaamaan repositoriosi ja push:aamaan sinne materiaalia.

Repositorion kloonaaminen

Kun olet githubissa katsomassa reposi sisältöä ja se on tyhjä näet siinä ohjeet miten committaa sinne materiaalia, mutta niistä kiinnostavia on seuraava tekstinpala git@github.com:<username>/<projektinNimi>.git
Nyt tiedät kaiken tarvittavan kloonataksesi repositorion. Kloonaaminen tapahtuu menemällä siihen kansioon, johon haluat git repon omaavan kansion syntyvän.
git clone git@github.com:<username>/<projektinNimi>.git
tämä luo kansion projektinNimi kansioon jossa parhaillaan olet. Mikäli teet tämän ennen kuin olet lisänny tavaraa repoosi saat virheilmoituksen, että olet kloonannut tyhjän repositorion. Siitä ei tarvitse häiriintyä.

Nyt siirry repositorioosi cd projektinNimi ja olet git:in omaavassa kansiossa. Siirrä haluamasi tiedostot tään kansioon, niin saat ne git:in toimintaalueelle. Voit esimerkiksi kopioida tähän kansioon NetBeansProjects kansion sisällön. Tai tietyn projektin kansiot. Huomaa, että joudut avaamaan tähän kansioon kopioimasi materiaalit NetBeanssisa uudestaan tästä kansiosta, jotta muokkaat versionhallinnan alaisia tiedostoja.

Voit myös hyödyntää gitHubin ohjeita. Ne ovat näppärämmät, mikäli sinulla on dataa kansiossa, mutta joudut gitHubiin pushatessa kirjoittelemaan enemman. Voit siis lisätä gitHubin jo olemassa olevan git repon remote repoksi, seuraavalla komennolla git remote add <kutsumanimi repolle > <git repon osoite> esimerkiksi git remote add gitHub git@github.com:username/appname.git Pushataksesi nyt committamasi muutokset githubiin joudut kirjoittamaan git push gitHub master
